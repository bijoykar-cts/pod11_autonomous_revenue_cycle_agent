Local demo boundary: local corpus validation is authoritative; persistence is disabled by default and is not production PHI compliance.

End-to-End Flow

1

Submit Note

Coder selects a synthetic case or pastes a single note. The UI keeps persistence off unless explicitly enabled.

2

Extract Evidence

The extraction agent finds diagnoses, procedures, negation, history-of phrasing, and documentation gaps.

3

Recommend Codes

Deterministic agents propose ICD-10-CM and ICD-10-PCS candidates with evidence offsets and redacted snippets.

4

Validate Locally

The validator checks the configured local ICD corpus and returns `valid` or `not_in_configured_corpus`.

5

Resolve Review State

Status is assigned as accepted, suggested, rejected, or needs documentation. Lower-confidence conflicts are suppressed.

6

Review Output

The workbench displays rationale, confidence components, flags, validation status, and safe debug trace IDs.

Safety Rules

  • No exact raw note text in validation errors or debug output.
  • Debug returns only trace ID and timing in local debug mode.
  • Unknown corpus versions fail with a safe error envelope.
  • Full note persistence is disabled by default.

Failure Modes

  • Negated and ruled-out conditions are flagged.
  • History-of and family-history language requires review.
  • Specificity gaps become documentation needs.
  • Absent corpus codes become `not_in_configured_corpus`.
